## Objetivo

Separar el código futuro en ensamblados con dependencias explícitas, impedir
dependencias circulares y mantener simulación y casos de uso aislados de Unity.

## Ensamblados implementados

| Assembly | Referencias | UnityEngine |
|---|---|---:|
| Domain | Ninguna | No |
| Application | Domain | No |
| Infrastructure | Domain, Application | Sí |
| Presentation | Domain, Application | Sí |
| Tests.EditMode | Producción y Test Runner | Sí |
| Tests.PlayMode | Producción y Test Runner | Sí |

## Grafo

```text
Domain
  ↑
Application
  ↑            ↑
Infrastructure Presentation
```

`Infrastructure` y `Presentation` no se referencian entre sí.

## Reglas vigentes

1. `Domain` no usa Unity.
2. `Application` no usa Unity.
3. `Infrastructure` no referencia `Presentation`.
4. `Presentation` no accede directamente a implementaciones de Infrastructure.
5. Producción no referencia tests.
6. Los scripts propios deben quedar bajo un `.asmdef`.
7. No se añaden referencias de paquetes sin uso real.
8. Los nombres de assemblies quedan congelados.
