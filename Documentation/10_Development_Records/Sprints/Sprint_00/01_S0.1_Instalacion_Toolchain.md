# S0.1 — Instalación y verificación del toolchain

## Resultado esperado

Una instalación funcional de Unity Hub y Unity Editor `6000.3.18f1`, con soporte para Windows y un IDE C# operativo.

## 1. Preparar ubicación de trabajo

Usar una ruta local corta, sin sincronización en nube y preferiblemente sin caracteres especiales.

Ejemplos recomendados:

```text
C:\Dev\CartridgeAndCloud
D:\UnityProjects\CartridgeAndCloud
```

Evitar:

```text
OneDrive
Dropbox
Google Drive
Escritorio sincronizado
Rutas de red
Carpetas históricas de Indie Studio Simulator
```

No crear todavía manualmente la carpeta final del proyecto si Unity Hub va a crearla.

## 2. Instalar o actualizar Unity Hub

1. Instalar la versión estable actual de Unity Hub.
2. Abrir Unity Hub.
3. Iniciar sesión con la cuenta de Unity.
4. Verificar que existe una licencia válida en:
   - `Settings > Licenses`, o equivalente según la versión del Hub.
5. Reiniciar el Hub si acaba de activarse la licencia.

### Evidencia

Captura recomendada:

```text
Evidence/S0.1_UnityHub_License.png
```

No incluir claves, correos privados ni identificadores sensibles en la captura.

## 3. Instalar Unity 6.3 LTS

1. En Unity Hub, abrir `Installs`.
2. Pulsar `Install Editor`.
3. Localizar la revisión exacta:
   - `Unity 6.3 LTS`
   - `6000.3.18f1`
4. Si no aparece en la lista principal, usar el archivo oficial de versiones y abrir la instalación mediante Unity Hub.
5. No instalar:
   - Unity 6.5.
   - Betas.
   - Alphas.
   - Revisiones distintas “por aproximación”.

## 4. Seleccionar módulos

Instalar como mínimo:

- Unity Editor.
- Windows Build Support.
- Windows Build Support (IL2CPP), si el Hub lo presenta como módulo separado.
- Microsoft Visual Studio Community, solo si no se utilizará Rider u otra instalación existente.

Opcional:

- Documentación local.
- Paquete de idioma del Editor.

No instalar aún:

- Android Build Support.
- iOS Build Support.
- Web Build Support.
- Linux Build Support.
- Dedicated Server.
- Herramientas de plataformas no contempladas por el vertical slice.

## 5. Verificar el Editor

1. En Unity Hub, comprobar que la instalación indica `6000.3.18f1`.
2. Abrir el Editor sin proyecto o mediante un proyecto temporal si fuera necesario.
3. Verificar en `Help > About Unity`:
   - versión `6000.3.18f1`;
   - arquitectura x64;
   - licencia activa.
4. Cerrar el Editor.

## 6. Configurar IDE

Elegir una única opción principal:

### Opción A — Visual Studio

- Visual Studio 2022.
- Workload: `Game development with Unity`.
- Componente de depuración para Unity habilitado.

### Opción B — JetBrains Rider

- Versión estable compatible con Unity 6.
- Plugin de Unity habilitado.

La integración definitiva se seleccionará dentro del proyecto en:

```text
Edit > Preferences > External Tools > External Script Editor
```

## Incidencias frecuentes

### La revisión exacta no aparece en Hub

Abrir la revisión desde el archivo oficial de Unity y elegir `Install with Unity Hub`.

### Package Manager o Hub no conecta

Comprobar:

- proxy;
- firewall;
- fecha y hora del sistema;
- certificados;
- sesión de Unity Hub;
- conectividad con el registro oficial de paquetes.

No añadir registros UPM de terceros para resolver este bloque.

### El módulo Windows no aparece

En `Installs`, abrir el menú de la instalación y usar `Add modules`.
