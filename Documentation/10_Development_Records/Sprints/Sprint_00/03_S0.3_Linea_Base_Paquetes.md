# S0.3 — Línea base de paquetes

## Principio de trabajo

Unity Package Manager debe resolver primero versiones compatibles con Unity `6000.3.18f1`.

No se copiará un `manifest.json` de otro proyecto y no se forzará manualmente una versión encontrada en Internet. La versión definitiva será la que Unity 6.3 muestre como **Released** y resuelva correctamente.

## Paquetes directos requeridos

| Paquete | ID técnico | Acción |
|---|---|---|
| Universal Render Pipeline | `com.unity.render-pipelines.universal` | Ya debe venir con la plantilla |
| Input System | `com.unity.inputsystem` | Instalar |
| AI Navigation | `com.unity.ai.navigation` | Instalar |
| Localization | `com.unity.localization` | Instalar |
| Test Framework | `com.unity.test-framework` | Conservar o instalar |
| IDE integration | `com.unity.ide.visualstudio` o Rider | Conservar la elegida |
| Unity UI | `com.unity.ugui` | Conservar si la plantilla lo incluye |

## Paquetes que no deben instalarse en este bloque

- Addressables.
- Cinemachine.
- ProBuilder.
- Splines.
- Recorder.
- Visual Scripting.
- Entities/DOTS.
- Netcode.
- Multiplayer.
- Steamworks.
- Newtonsoft JSON.
- Unity Gaming Services.
- Analytics.
- Ads.
- IAP.
- Cloud Save.
- Asset Store tools.
- Paquetes Preview o Experimental.

La presencia de algún paquete de plantilla no implica que deba eliminarse inmediatamente. Primero se audita su función y dependencia.

## 1. Abrir Package Manager

```text
Window > Package Management > Package Manager
```

Según el layout del Editor puede aparecer simplemente como:

```text
Window > Package Manager
```

Seleccionar:

```text
Packages: Unity Registry
```

## 2. Verificar URP

1. Buscar `Universal RP`.
2. Confirmar estado instalado.
3. Confirmar que no es una versión pre-release.
4. No pulsar Update si Package Manager propone una versión que cambia de línea compatible.
5. Registrar la versión exacta.

Para Unity 6.3 se espera una línea URP propia de esa versión del Editor; la revisión exacta debe tomarse del proyecto, no de una tabla externa.

## 3. Instalar Input System

1. Buscar `Input System`.
2. Seleccionar la versión marcada como Released y recomendada por Package Manager.
3. Pulsar `Install`.
4. Si Unity pregunta si debe activar el nuevo backend, aceptar.
5. Reiniciar el Editor cuando lo solicite.
6. Ir a:

```text
Edit > Project Settings > Player > Other Settings > Configuration
```

7. Configurar:

```text
Active Input Handling: Input System Package (New)
```

8. Reiniciar de nuevo si Unity lo requiere.
9. Confirmar que no quedan errores de compilación.

No seleccionar `Both` salvo diagnóstico temporal documentado.

## 4. Instalar AI Navigation

1. Buscar `AI Navigation`.
2. Instalar la versión Released seleccionada por Unity.
3. No importar samples.
4. Confirmar la aparición de los componentes del paquete.
5. No crear todavía NavMeshSurface ni agentes.

## 5. Instalar Localization

1. Buscar `Localization`.
2. Instalar la versión Released seleccionada por Unity.
3. No crear todavía:
   - Localization Settings;
   - Locales;
   - String Tables;
   - Asset Tables.
4. No importar samples.

La configuración ES/EN se realizará en un bloque posterior cuando exista estructura de contenido.

## 6. Verificar Test Framework

1. Buscar `Test Framework`.
2. Si está instalado, conservarlo.
3. Si no está instalado, instalar la versión Released recomendada.
4. Confirmar que existe:

```text
Window > General > Test Runner
```

No crear aún tests; se crearán en S0.8.

## 7. Revisar integración de IDE

Conservar una sola integración principal:

- `Visual Studio Editor`, o
- integración de Rider.

No es obligatorio eliminar otras integraciones si son dependencias inofensivas del Editor, pero deben registrarse.

## 8. Auditoría de paquetes

Abrir:

```text
Packages/manifest.json
Packages/packages-lock.json
```

Comprobar:

- ambos archivos existen;
- JSON válido;
- los cinco paquetes requeridos están resueltos;
- no hay versiones `-pre`, `-exp`, `-preview` en paquetes requeridos;
- no hay scoped registries de terceros;
- no hay referencias `file:`, Git o rutas locales;
- las versiones del lock corresponden al manifest y sus dependencias.

No modificar `packages-lock.json` manualmente.

## 9. Capturar la línea base

Cerrar Unity antes de copiar o calcular hashes.

Ejecutar desde la raíz del proyecto:

```powershell
PowerShell -ExecutionPolicy Bypass -File .\Tools\Collect-S0Evidence.ps1
```

Si la carpeta `Tools` todavía está fuera del proyecto, copiarla temporalmente a la raíz del repositorio/proyecto antes de ejecutar.

El informe se generará en:

```text
Evidence/S0.1-S0.3_Automated_Report.md
```

## 10. Política ante versiones diferentes

Si la versión instalada no coincide con una expectativa externa pero cumple todo lo siguiente:

- aparece como Released;
- Package Manager la ofrece para Unity 6.3;
- resuelve sin errores;
- no es Preview/Experimental;
- el proyecto reabre correctamente;

entonces se acepta y se registra como línea base.

No actualizar paquetes después de congelar el bloque sin una entrada de trazabilidad.

## 11. Criterios de aceptación S0.3

- [ ] URP instalado y activo.
- [ ] Input System instalado.
- [ ] Active Input Handling = New.
- [ ] AI Navigation instalado.
- [ ] Localization instalado.
- [ ] Test Framework instalado.
- [ ] No se importaron samples.
- [ ] No hay paquetes requeridos pre-release.
- [ ] No hay registros de terceros.
- [ ] `manifest.json` válido.
- [ ] `packages-lock.json` válido.
- [ ] Proyecto reabre sin errores.
- [ ] Informe automático generado.
