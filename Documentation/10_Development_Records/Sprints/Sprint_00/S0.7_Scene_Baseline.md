# S0.7 — Scene Baseline

**Proyecto:** Cartridge & Cloud  
**Sprint:** 0  
**Bloque:** S0.7  
**Estado:** In Progress  
**Editor:** Unity `6000.3.18f1`  
**ADR:** ADR-0005  
**Último commit de partida:** `31a77be358a3b8e9268cd69abb02a1f52d429fbd`

## 1. Objetivo

Crear, guardar, registrar y validar las cuatro escenas base del proyecto sin
añadir lógica, navegación o contenido jugable.

## 2. Preparación

Antes de abrir Unity:

1. Confirmar que GitHub Desktop no tiene cambios anteriores pendientes.
2. Integrar la documentación y los XLSX v0.3.2 de este paquete.
3. Eliminar los tres XLSX v0.3.1 indicados en `FILES_TO_REMOVE_S0.7.txt`.
4. Eliminar desde el Explorador de Windows:

```text
Assets/_Project/Scenes/.gitkeep
Assets/_Project/Scenes/.gitkeep.meta
```

El segundo archivo solo se elimina si existe.

5. No eliminar `Assets/_Project/Scenes.meta`.
6. Abrir Unity mediante Unity Hub con `6000.3.18f1`.
7. Esperar a que finalice la importación.
8. Confirmar cero errores rojos antes de crear escenas.

## 3. Crear Bootstrap

1. Seleccionar `File > New Scene`.
2. Elegir `Empty` si está disponible.
3. Si solo aparece `Basic (URP)`, crearla y eliminar todos sus GameObjects.
4. La jerarquía debe quedar completamente vacía.
5. Guardar mediante `File > Save As` en:

```text
Assets/_Project/Scenes/Bootstrap.unity
```

No añadir cámara, luz, managers, scripts ni objetos persistentes.

## 4. Crear MainMenu

1. Seleccionar `File > New Scene`.
2. Elegir `Basic (URP)`.
3. Conservar únicamente:

```text
Main Camera
Directional Light
```

4. Eliminar cualquier objeto extra generado por la plantilla, como un volumen
   global, si apareciera.
5. No modificar los valores de la plantilla salvo que Unity lo requiera.
6. Guardar como:

```text
Assets/_Project/Scenes/MainMenu.unity
```

No crear Canvas, EventSystem, botones o localización.

## 5. Crear Store

Repetir el procedimiento de `MainMenu` y guardar como:

```text
Assets/_Project/Scenes/Store.unity
```

La jerarquía debe contener únicamente:

```text
Main Camera
Directional Light
```

No crear suelo, parcela, jugador, tienda, cámara orbital, cuadrícula o NavMesh.

## 6. Crear TestLab

Repetir el procedimiento y guardar como:

```text
Assets/_Project/Scenes/TestLab.unity
```

La jerarquía debe contener únicamente:

```text
Main Camera
Directional Light
```

No añadir prototipos todavía.

## 7. Retirar SampleScene

Si existe:

```text
Assets/Scenes/SampleScene.unity
```

1. Abrirla y confirmar que no contiene trabajo propio.
2. Eliminarla desde la ventana Project de Unity.
3. Si `Assets/Scenes` queda vacía, eliminar también la carpeta desde Unity.
4. No eliminar `Assets/Settings`, render pipeline assets ni configuraciones URP.

## 8. Configurar la Scene List global

1. Abrir `File > Build Profiles`.
2. Entrar en `Platforms > Scene List`.
3. Eliminar `SampleScene` de la lista si aparece.
4. Arrastrar las cuatro escenas desde Project.
5. Habilitar las cuatro entradas.
6. Ordenarlas exactamente:

```text
0  Bootstrap
1  MainMenu
2  Store
3  TestLab
```

7. No crear todavía un Build Profile personalizado.
8. No activar `Override Global Scene List`.

El cambio normalmente se registra en:

```text
ProjectSettings/EditorBuildSettings.asset
```

## 10. Validación Play Mode

Para cada escena:

1. Abrirla individualmente.
2. Guardar con `Ctrl+S`.
3. Entrar en Play Mode.
4. Esperar al menos cinco segundos.
5. Salir de Play Mode.
6. Revisar la consola.

En `Bootstrap` es normal no ver una cámara en Game View. No es aceptable que
aparezca un error rojo.

Después:

1. Cerrar Unity.
2. Abrirlo de nuevo.
3. Confirmar que las cuatro escenas siguen presentes.
4. Confirmar que la Scene List conserva el orden.
5. Confirmar que la consola sigue limpia.
