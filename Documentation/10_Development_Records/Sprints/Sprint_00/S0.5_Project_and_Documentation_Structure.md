# S0.5 — Project and Documentation Structure

**Proyecto:** Cartridge & Cloud  
**Sprint:** 0  
**Bloque:** S0.5  
**Estado inicial:** pendiente de ejecución local

## Objetivo

Establecer una estructura estable para los assets propios del juego y para toda la
documentación oficial y operativa del proyecto.

## Alcance

- Crear `Assets/_Project`.
- Crear categorías iniciales de arte, audio, contenido, prefabs, escenas, código,
  configuración, tests y UI.
- Incorporar el paquete documental oficial v0.3 dentro del repositorio.
- Separar línea base oficial, registros de desarrollo, borradores y archivo.
- Mantener `Evidence` fuera de `Documentation` y fuera de `Assets`.

## Fuera de alcance

- Assembly Definitions.
- Escenas definitivas.
- Código de gameplay.
- Movimiento de assets URP de la plantilla.
- Eliminación de la escena de muestra.
- ADRs completos.
- Configuración de Git LFS.

## Definition of Done

- [ ] Existe `Assets/_Project`.
- [ ] Las carpetas previstas están creadas.
- [ ] El paquete oficial está descomprimido en `Documentation/00_Official_Baseline/v0.3`.
- [ ] El baseline contiene 61 archivos.
- [ ] No se ha incluido el RAR original.
- [ ] Los registros del Sprint 0 están en `Documentation/10_Development_Records`.
- [ ] `Evidence` permanece en la raíz del repositorio.
- [ ] Unity abre sin errores.
- [ ] GitHub Desktop no muestra `Library`, `Logs` ni `UserSettings`.
- [ ] Los cambios han sido revisados, confirmados y enviados a `main`.

## Criterios de aceptación

### AC-S0.5-001

Todos los assets propios futuros tienen una raíz única: `Assets/_Project`.

### AC-S0.5-002

Los documentos oficiales pueden localizarse sin entrar en Unity.

### AC-S0.5-003

La estructura original del paquete v0.3 se conserva bajo la línea base.

### AC-S0.5-004

Una clonación nueva recupera la estructura mediante los archivos `.gitkeep`.

### AC-S0.5-005

No existen documentos oficiales duplicados en la raíz del repositorio.
