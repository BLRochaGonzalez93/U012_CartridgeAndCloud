# 1. Objetivo

Dar legibilidad a acciones y ambiente sin fatiga.

# 2. Eventos implementados de Fase 1

Ambience Store, Music Store, UI Confirm/Error, Placement Valid/Invalid, Order Received, Door, Checkout y Day Closed.

# 3. Reglas

- feedback corto y distintivo;
- mezcla que preserve UI y eventos;
- loops sin clicks;
- volumen configurable;
- no depender del audio como único feedback;
- clips originales o licenciados.

# 4. Validación pendiente S16/S17

Niveles relativos, repetición, espacialización de puerta/checkout, transiciones de día, mute/volumen y build externa.
